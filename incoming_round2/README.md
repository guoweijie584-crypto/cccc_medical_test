# 第二次汇报材料投递区

后面你要补给我的新增材料，统一先放到这个目录。

## 你可以直接放什么

- 新增的 `pptx` / `pdf` / `docx`
- 截图、图片、表格
- 你这段时间记录的零散想法
- 第一次汇报后的反馈
- 你认为第二次汇报要重点补的内容

## 最省事的放法

### 1. 原始文件

直接把文件丢进这个目录就行。

如果文件很多，建议你按下面分子目录：

- `slides/`
- `docs/`
- `images/`
- `notes/`
- `feedback/`

不分也可以，我后面可以再整理。

### 2. 零散想法 / 口头补充

如果是文字想法，直接新建一个 `md` 或 `txt` 文件放这里即可。

推荐文件名：

- `notes/第二次汇报-你想到的点.md`
- `feedback/第一次汇报后反馈.md`
- `notes/第二次汇报想补的内容.md`

## 文字内容最好怎么写

不用写得很正式，按下面这种最省事：

```md
## 这条信息是什么

- 来源：老师 / 同学 / 自己复盘 / 新增文件
- 时间：大概什么时候
- 结论：一句话先说重点
- 影响：它更像是
  - 要补进第二次汇报主线
  - 只是参考材料
  - 需要后续核验

## 详细内容

这里随便写，想到什么写什么。
```

## 传输方式

你和我在同一个工作区里，所以最简单的方式不是额外发链接，而是：

1. 直接把文件放进 `incoming_round2/`
2. 然后在聊天里告诉我：
   - 哪些是新增文件
   - 哪些是你自己的想法/反馈
   - 哪些你觉得优先级最高

## 我后面会怎么处理

我会按下面顺序接：

1. 先做文件归档和来源判断
2. 给可读文件补 sidecar 文本
3. 区分“第二次主线候选” / “参考材料” / “反馈待核验”
4. 再决定哪些内容进入第二次汇报规划文档

## 一条原则

先把原始材料都丢进来，不用先替我整理成最终答案。

你先尽量保留原始信息，我来负责后面的归档、筛选和整理。

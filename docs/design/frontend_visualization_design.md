# 前端可视化设计方案

> 状态：设计方案 | 日期：2026-04-07

## 一、总体架构

### 在 MedicalTab 中新增两个视图

现有视图：`dashboard | list | detail | evolution | config`

新增：
- **`memory`** — 记忆宫殿可视化（重点，动效精美）
- **`evaluation`** — 人工评价管理界面

### 技术选型

| 需求 | 方案 |
|------|------|
| 动效引擎 | **framer-motion** — React 生态最成熟的动效库 |
| 树形/力导向图 | **CSS + framer-motion** 手写（不引入 D3 重量级依赖） |
| 粒子/光效 | **CSS animations + canvas** |
| 状态管理 | 复用现有 **zustand** |
| API 调用 | 复用现有 `medicalApiUrl()` |

## 二、记忆宫殿可视化（Memory Palace View）

### 设计理念："上帝视角管理记忆"

视觉风格：
- **深色太空背景** — 深蓝/紫渐变，微弱星尘粒子
- **记忆节点是发光星球** — 不同类别不同颜色光晕
- **连接线是能量脉络** — 父子节点间有流动光线
- **Vitality 通过脉搏表达** — 高活力节点明亮脉动，低活力暗淡
- **操作有华丽反馈** — 创建记忆有"凝聚"动画，删除有"消散"粒子

### 页面布局

```
┌─────────────────────────────────────────────────────┐
│  Memory Palace · 记忆宫殿                    [搜索🔍] │
├────────────┬────────────────────────────────────────┤
│            │                                        │
│  患者列表   │     记忆星图 (中央可视化区域)            │
│            │                                        │
│  P001 ●    │      ★ profile                         │
│  P002 ○    │     / | \                              │
│  P003 ○    │  ★   ★   ★                            │
│            │ glucose medication diet                │
│            │                                        │
│            ├────────────────────────────────────────┤
│            │  记忆详情面板（选中节点时展开）           │
│            │  内容 | 优先级 | 触发条件 | 操作按钮     │
└────────────┴────────────────────────────────────────┘
```

### 记忆节点视觉映射

| 属性 | 视觉表达 |
|------|---------|
| 类别 (category) | 颜色：profile=金色, glucose=青色, medication=蓝色, diet=绿色, safety=红色 |
| Priority | 大小：priority 0 最大，5 最小 |
| Vitality | 亮度 + 脉搏频率：高活力=明亮快速脉动，低活力=暗淡慢速 |
| 最近访问 | 外圈光环：刚被检索的节点有扩散光环 |

### 交互

1. **点击节点** — 展开详情面板，显示完整内容、元数据
2. **悬停节点** — 显示 tooltip（类别、priority、摘要）
3. **搜索** — 匹配节点高亮闪烁，其余暗淡
4. **新建记忆** — 弹窗填写，提交后节点从中心"凝聚"飞入位置
5. **删除记忆** — 节点"碎裂消散"动画
6. **拖拽调整** — 未来扩展

### 组件拆分

```
pages/medical/
├── MemoryPalaceView.tsx       — 主视图容器
├── MemoryStarMap.tsx           — 中央星图可视化
├── MemoryNode.tsx              — 单个记忆节点（发光球体）
├── MemoryDetailPanel.tsx       — 选中节点的详情面板
├── MemorySearchBar.tsx         — 搜索栏
├── PatientMemorySidebar.tsx    — 左侧患者选择
└── MemoryCreateModal.tsx       — 新建记忆弹窗
```

## 四、需要新增的后端 API

记忆可视化需要从 Memory Palace 读取树形结构，现有 api_server.py 只有 `/api/patients/{id}/memories`（扁平列表）。

新增：
```
GET /api/memory/tree/{patient_id}     — 患者记忆树（含子节点层级）
GET /api/memory/search                — 搜索记忆（代理到 Memory Palace）
POST /api/memory/create               — 创建记忆节点
PUT /api/memory/{path}                — 更新记忆节点
DELETE /api/memory/{path}             — 删除记忆节点
GET /api/memory/stats                 — 记忆统计
```

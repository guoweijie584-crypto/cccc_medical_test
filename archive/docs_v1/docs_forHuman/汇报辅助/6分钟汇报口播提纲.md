# 6 分钟汇报口播提纲

## 这份文档怎么用

这份文档是给明天汇报直接用的。

你可以把它理解成：

- 该保留哪几页
- 每页讲什么
- 每页大概讲多久
- 哪些词必须收紧

默认边界：

- 当前只有文档证据，没有源码入库证据
- 可以讲“系统设计、框架成型、路线明确”
- 不能讲“完整落地、稳定运行、闭环已跑通”

## 推荐页序

建议主讲 6 页，加封面和致谢即可。

### 封面

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 1 页

用途：

- 简洁引入
- 明确这是阶段汇报，不是最终答辩

口播建议：

`各位老师好，我们汇报的项目是“面向智能客服场景的 AI Agent 血糖管理自进化系统”。这次汇报重点不是展示完整成品，而是介绍我们当前已经形成的系统框架、记忆管理思路和后续推进方向。`

建议时长：

- 15 秒

## 第 1 页：项目背景

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 4 页

必须讲的 3 个点：

- 血糖管理不是一次性问答，而是长期连续管理问题
- 这个场景天然需要长期上下文和多角色协作
- 所以多 Agent + 记忆管理是场景驱动，不是技术堆砌

口播建议：

`我们选择血糖管理场景，是因为它天然需要长期上下文、历史信息调用和多角色协作。传统管理方式往往难以做到持续跟踪和实时响应，因此我们希望用多 Agent 协作和记忆管理来提升连续服务能力。`

建议时长：

- 45 秒

## 第 2 页：系统整体架构

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 5 页

必须讲的 3 个点：

- 主治医生 Agent 负责统筹与最终输出
- 药剂师、营养师、代谢病医生等专家负责分工协作
- 记忆管理 Agent 负责三层记忆的统一治理

口播建议：

`在系统设计上，我们不是简单并列多个 Agent，而是形成了一个有主次关系的协作体系。主治医生 Agent 负责统筹与最终决策，专家 Agent 负责分工分析，记忆管理 Agent 负责提供和维护长期上下文支持。`

收紧提醒：

- 不要展开 MCP、Ledger、Daemon、CLI、IM Bridge 这些底层词
- 这页只讲业务架构，不讲运行全景图

建议时长：

- 55 秒

## 第 3 页：业务流程图

来源：

- `archive/round1_baseline/流程架构.pptx` 第 1 页

必须讲的 3 个点：

- 用户问题先进入 Primary
- 系统先取回记忆，再并行咨询专家
- 最后由主 Agent 汇总输出，并把关键事实写回记忆

口播建议：

`一次典型咨询的流程是：用户提出问题后，主 Agent 先判断需求与风险，再调用记忆模块获取相关历史信息，然后并行咨询不同专家，最后统一整合成面向用户的建议，并把本轮关键事实写回记忆系统。`

补充点：

- 多 Agent 的价值在于专业分工，而不是角色堆砌
- 对外输出保持统一口径，不暴露内部协作细节

建议时长：

- 60 秒

## 第 4 页：记忆管理设计

来源：

- `archive/round1_baseline/流程架构.pptx` 第 2 页

必须讲的 3 个点：

- 三层记忆是当前最核心的创新主线
- 短期、中期、长期分别对应不同时间尺度的信息
- Memory Agent 的定位是治理记忆，而不是做业务决策

口播建议：

`我们当前最核心的设计重点是三层记忆管理。短期记忆服务当前对话，中期记忆沉淀阶段性趋势，长期记忆保留用户画像和病史等稳定信息。我们希望通过独立的 Memory Agent 来统一治理这些信息，让系统具备连续服务能力。`

收紧提醒：

- TTL、数据库选型、向量索引都不要讲成定版实现
- 可以说“当前设计示意中考虑了分层存储和语义检索”

建议时长：

- 60 秒

## 第 5 页：当前阶段进展

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 8 页

必须讲的 3 个点：

- 已完成方向定义与整体架构表达
- 已完成角色分工和三层记忆分层设计
- 当前阶段是“框架成型、关键机制推进中”

口播建议：

`当前阶段，我们已经完成了项目方向定义、多 Agent 协作架构设计、角色分工表达，以及三层记忆的分层设计与初步方案。也就是说，系统框架已经成型，但关键机制仍处于继续推进和做实的阶段。`

收紧提醒：

- `多 Agent 架构及代码实现` 口头上改成 `多 Agent 协作框架的初步搭建`
- `初步配置` 可以改成 `初步方案` 或 `初步设计`

建议时长：

- 55 秒

## 第 6 页：下一阶段推进

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 9 页

必须讲的 3 个点：

- 下一阶段重点 1：把记忆真正接入系统流程
- 下一阶段重点 2：细化评测、归因与优化机制
- 下一阶段重点 3：逐步打通 memory + prompt 的双闭环

口播建议：

`下一阶段我们会重点推进两件事。第一，是把记忆模块从分层设计推进到真实接入，形成更稳定的读取、写入和更新机制。第二，是把自进化从总体框架推进到可验证的评测、归因和优化流程，逐步打通 memory 和 prompt 的双闭环。`

收紧提醒：

- 不要说“即将完成”
- 不要说“已经实现，只差展示”

建议时长：

- 55 秒

## 致谢页

来源：

- `archive/round1_baseline/3.26-SR-14.11.pptx` 第 10 页

口播建议：

`以上就是我们的阶段性汇报，感谢各位老师和同学。`

建议时长：

- 10 秒

## 总结版一句话口径

如果全场只能记住一句话，建议统一成：

`我们目前已经把多 Agent 协作框架和三层记忆管理的核心思路搭起来了，下一阶段重点是把记忆接入和自进化机制进一步做实。`

## 明天最需要避免的说法

- 我们已经完整实现了系统
- 我们已经完成双闭环
- 系统已经稳定自进化
- 这些参数已经最终定版
- 这套机制已经跑出了确定效果

## 替代表达

- 已完成框架设计
- 已完成初步搭建
- 已形成总体路线
- 正在推进关键机制
- 下一阶段重点补齐

2. 如果我要快速理解项目，应该按什么顺序读

## 二、新文件以后统一怎么接入

后面你再往这个目录里加新文件，我会按下面 5 步处理，而不是直接把它们写进主结论。

### 第一步：先生成可比较文本

如果新进来的是 PDF / PPTX，先看它有没有对应的 sidecar 文本：

- `tmp/extracted_text/<原文件名>.txt`

没有的话，先补这一层，再进入后续比较流程。

原因很简单：

- 没有 sidecar 文本，就很难稳定比较多版稿件
- 每来一份新文件都重新人工开二进制文件比，流程不可复用

### 第二步：先按文件性质挂到暂存类别

我会先把新文件挂到下面几类之一：

- `课程要求`
- `汇报主稿候选`
- `机制图`
- `参考/展示材料`
- `源码/运行证据`
- `临时噪音文件`
- `待判定`

如果它被挂到 `汇报主稿候选`，默认含义不是“和所有旧稿泛泛比较”，而是：

- 先与当前基线 `3.26-SR-14.11.*` 比
- 只有比完基线后，才讨论它是否升级为新的主稿线

### 第三步：只根据已核对证据决定是否升级身份

比如：

- 只有在比对后，某个新 PPTX 才能从“汇报主稿候选”升级成“当前主稿”
- 只有在确认它主要承担流程说明时，某个新文件才能升级成“机制图”
- 只有在确认它只是展示模板或视觉稿时，才会进入“参考/展示材料”

如果它当前属于 `待判定`，最少先回答下面 3 个问题：

1. 它是不是现有主稿线的同源改版
2. 它是不是展示稿 / 模板 / 视觉稿
3. 它有没有新增、且会改写当前主线结论的内容

只有这 3 个问题至少有了初步答案，`待判定` 文件才允许继续升级身份。

### 第四步：canonical 文档不会被新文件直接改写

新文件进来之后，先更新的是：

- `资料分类与阅读顺序.md`
- `README.md`
- `docs_forAI/01_source_index.md`

只有在完成比对并确认“它改变了主线判断”之后，才会去动：

- `项目梳理与理解.md`
- `系统设计与模块设计.md`
- `阶段进度与里程碑.md`

### 第五步：阶段入口页只接收结论，不接收未判定原料

也就是说：

- `阶段入口/第一次汇报.md`
- `阶段入口/第二次汇报准备.md`
- `阶段入口/最终汇报准备.md`

这几页不会直接堆新文件名单。

它们只会接收已经经过比对后的稳定结论，避免入口页被原始材料池污染。

## 五、证据不要单一排序，要按问题类型判断

如果你更关心“这句话到底凭什么成立”，不要把所有问题都套进同一份排序里。

更稳的做法是按问题类型分：

### 1. 你在问“现在到底实现到了哪一步”

优先看：

1. `tmp/source_repo/cccc_medical_test/`
2. `../docs_forAI/03_system_reference.md`
3. `../docs_shared/证据边界与事实口径.md`

这里的核心原则是：

- 讲实现状态，源码证据优先
- 但“有源码”仍不自动等于“已稳定验证”

### 2. 你在问“第一次汇报主线该怎么说”

优先看：

1. `3.26-SR-14.11.*`
2. `archive/round1_baseline/流程架构.pptx`
3. `archive/requirements_and_constraints/Multi Agent框架-0326.pdf`
4. `../docs_shared/证据边界与事实口径.md`

这里的核心原则是：

- 讲当前汇报主线，主稿、机制图和策略口径优先
- 不要反过来让源码结构去主导第一次汇报的讲法

### 3. 你在问“课程要求细项到底是什么”

理论上优先看：

1. `archive/requirements_and_constraints/3.九安医疗-课程选题信息.pdf`

但当前必须同时记住：

- `tmp/extracted_text/3.九安医疗-课程选题信息.pdf.txt` 仍然是坏的
- 所以任务书细项虽然理论上优先级最高，当前却仍然要按“待复核”处理

### 4. 你在问“这句话该写多重”

优先看：

1. `../docs_shared/证据边界与事实口径.md`

然后再判断它属于：

- `代码已证实`
- `文档已明确`
- `设计级细节`
- `待任务书复核`

## 七、阶段视角下的最短理解

如果你现在是按“第一次汇报 / 第二次汇报 / 最终汇报”来理解材料，那么最稳的判断是：

- 第一次汇报材料最完整，已经有比较成熟的主稿线
- 第二次汇报更适合建立在“系统设计 + 进度 + 源码证据”上，但当前还没独立收口成稿
- 最终汇报需要更多结果、验证和最终展示证据，当前目录里也还没有独立最终稿

### 如果你的目标是“核对原始依据”

建议顺序：

1. `docs_forAI/01_source_index.md`
2. `docs_forAI/02_extraction_status.md`
3. `tmp/extracted_text/`
4. 原 PDF / PPTX / 源码仓库

# 明日汇报 PPT 建议稿

## 这份文档怎么用

这份文档不是最终 PPT 文件，而是一版可以直接拿去改 PPT 的结构稿。

目标：

- 适配明天的阶段汇报
- 控制在 6 分钟左右
- 重点讲清“为什么做、系统怎么设计、现在做到哪、接下来怎么推”
- 不把路线图讲成已实现

建议页数：

- 6 页到 7 页最稳

## 总体讲述逻辑

建议整场汇报只围绕 4 个问题展开：

1. 为什么这个选题成立
2. 系统是怎么设计的
3. 当前阶段已经做了什么
4. 接下来重点补什么

## 版本 A：6 页精简版

### 第 1 页：项目背景与选题理由

标题建议：

`项目背景：为什么选择血糖管理场景`

页面要点：

- 糖尿病患者多，长期管理需求强
- 传统管理不连续，难以 7×24 响应
- 血糖管理天然需要长期上下文、角色协作和历史信息调用

讲述重点：

- 不是为了做一个复杂的 Agent 框架
- 而是因为这个场景本身就需要多角色协作和记忆支持

一句话讲法：

`我们选择血糖管理，不是为了炫技，而是因为这个场景天然适合用多 Agent 协作和长期记忆管理来解决。`

### 第 2 页：系统整体架构

标题建议：

`系统设计：多 Agent 协作架构`

页面要点：

- 主治医生作为唯一主要决策者
- 药剂师、营养师、代谢病医生作为辅助专家
- 科普老师、心理咨询师作为拓展角色
- 记忆管理 Agent 独立治理三层记忆

讲述重点：

- 主治医生负责整合与最终输出
- 专家负责分工分析
- Memory Agent 不做业务决策，而是负责上下文治理

一句话讲法：

`我们的系统不是多个 Agent 并排放着，而是一个有主次分工、有记忆治理的协作体系。`

### 第 3 页：业务流程与记忆参与

标题建议：

`业务流程：记忆如何参与多 Agent 决策`

推荐图源：

- 直接用 `archive/round1_baseline/流程架构.pptx` 的业务流程图

页面要点：

- 用户发起咨询
- Primary 调用三层记忆
- 并行咨询专家
- 整合输出给用户
- 写回关键事实

讲述重点：

- 记忆不是附属模块，而是协作前和协作后的关键环节
- 这也解释了为什么记忆管理必须独立出来

一句话讲法：

`记忆在这里不是被动存档，而是参与协作决策的核心支撑模块。`

### 第 4 页：核心创新点

标题建议：

`核心设计：三层记忆管理 + 自进化路线`

页面要点：

- 创新点 1：三层记忆管理
- 创新点 2：面向 memory + prompt 的自进化推进路线

讲述重点：

- 当前最核心的创新主线是记忆管理
- 自进化是后续重点推进方向
- 自进化不是简单改 prompt，而是先评测、再归因、再分别优化

一句话讲法：

`我们的重点不是把自进化讲成已经跑通，而是说明我们已经把它设计成一个可继续推进的明确闭环。`

### 第 5 页：当前阶段进展

标题建议：

`阶段进展：我们现在已经完成了什么`

页面要点：

- 已完成项目方向定义
- 已完成多 Agent 架构与角色分工设计
- 已完成三层记忆框架设计
- 已完成自进化路线设计

讲述重点：

- 当前已经完成的是框架、结构和路线
- 当前还没有源码证据进入这份工作区，所以不讲成“系统全面落地”

一句话讲法：

`当前阶段我们已经把系统框架搭起来了，重点是把关键机制继续做实。`

### 第 6 页：下一阶段规划

标题建议：

`下一阶段：从框架设计走向关键机制落地`

页面要点：

- 记忆模块真正接入系统流程
- 完善记忆写入、整合与更新机制
- 细化评测、归因与优化机制
- 形成可验证的闭环优化过程

讲述重点：

- 这页一定要讲成推进路线
- 不能讲成“已经完成，只是还没展示”

一句话讲法：

`下一阶段我们会优先把记忆治理和评测驱动优化做实，让系统从架构表达进一步走向可验证能力。`

## 版本 B：7 页稳妥版

如果时间允许，可以在第 2 页和第 3 页之间加一页“角色与权限设计”。

### 可加页：角色与权限设计

标题建议：

`角色设计：主次分工与协作边界`

页面要点：

- Foreman 唯一主要决策
- 专家可提出质疑与补充建议
- 拓展角色提供非治疗类支持
- Memory Agent 负责统一管理记忆

这页的价值在于：

- 帮老师更快理解为什么你们不是“角色堆砌”
- 体现系统治理和协作设计是认真做过的

## 明天汇报时最需要收紧的词

下面这些词要谨慎：

- 已实现
- 已跑通
- 已落地
- 自动优化完成
- 完整闭环

建议替换成：

- 已完成设计
- 已完成初步搭建
- 已形成框架
- 已提出机制
- 下一阶段推进

## 不建议放太重的内容

为了保证 6 分钟内老师能抓住重点，以下内容不建议作为主线展开：

- Daemon、Ledger、IM Bridge、CLI、MCP 等过底层的系统细节
- 胶质细胞、记忆宫殿、睡眠重放等脑科学类比
- 太满的商业化叙事
- 没有源码或 demo 支撑的性能、效果、产品化结论

这些内容可以：

- 留作问答时补充
- 作为未来扩展方向一笔带过

## 最后一页收口话术

汇报结尾建议用这一类表达：

`我们当前已经完成了系统框架、角色协作、记忆分层和自进化路线的设计，下一阶段会重点把记忆治理与评测驱动优化进一步做实，让系统从“架构成型”走向“关键机制可验证”。`

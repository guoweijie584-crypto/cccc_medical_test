# 项目梳理与理解

## 这批文件在讲什么

这个目录不是代码仓库，而是一组围绕同一个课程项目逐步演化出来的材料。结合文件内容来看，你们在做的项目可以概括为：

`面向智能客服场景的 AI Agent 血糖管理自进化系统`

核心不是“做一个会聊天的医疗机器人”，而是想做一个：

- 有多角色协作的血糖管理系统
- 有三层记忆管理
- 能基于评测结果持续优化 prompt 和 memory 的系统

如果只用一句话概括，就是：

`用多智能体去模拟血糖管理团队，再把记忆管理和自进化机制做成这个项目的创新主线。`

## 这个项目目前最像什么

结合全部文件，我认为你们现在做的不是一个普通课程作业 PPT，而是一个比较明确的研究型原型项目。结构上已经相当清楚：

### 主线一：多 Agent 协作

稳定出现的角色有：

- 主治医生 / Primary / Foreman
- 药剂师
- 营养师
- 代谢病医生
- 科普老师
- 心理咨询师
- 记忆管理 Agent

其中主治医生是总负责人，其它角色按专业分工协作。

### 主线二：三层记忆管理

这是你们最稳定、也最有辨识度的创新点。

三层记忆通常被表述为：

- 短期记忆：当前对话、最近交互、本轮专家输出
- 中期记忆：近期趋势、最近药物调整、近期饮食运动情况
- 长期记忆：用户画像、病史、过敏史、重大事件

从现有材料看，这部分“定义”已经比较清楚，但“机制”还没有完全讲透。

### 主线三：自进化闭环

你们的自进化并不是只改 prompt，而是想做双闭环：

- Prompt 自进化
- Memory 自进化

典型链路是：

- Evaluator 质检
- Analyzer 归因
- Prompt Optimizer / Memory Optimizer 优化
- 重新评测

这个方向和现有汇报材料高度一致，也是后面最值得继续做实的部分。至于它和任务书是否逐条一致，仍需任务书复核。

## 目前目录里最值得统一的地方

我看到一个需要你们后续统一口径的问题：

### 1. “当前进度”和“愿景表达”之间有张力

`technology.pptx` 里的表达比较满，像成品。

但：

- 现有汇报策略文件明确要求按阶段推进来讲
- `archive/requirements_and_constraints/Multi Agent框架-0326.pdf` 也明确提醒不要把未完成的部分讲成已实现

所以后面不管是写 PPT、写文案，还是做 Demo 说明，建议统一成：

- 已完成什么
- 已设计但未完全实现什么
- 下一阶段要补什么

### 2. 记忆机制是核心，但还需要更具体

从材料里能看出你们已经非常确定“记忆管理”是核心创新点。  
但如果老师继续追问，最容易卡住的地方也会是：

- 记忆写入规则是什么
- 三层记忆怎么迁移
- 什么时候遗忘
- 哪些信息共享，哪些信息隔离

这部分后面应该优先补。

### 3. 自进化要避免讲成“自动改一切”

现有文件里比较好的表述是：

- 先评测
- 再归因
- 区分 prompt 问题和 memory 问题
- 分别优化

这个说法是稳的。  
如果讲成“系统会自动学习并不断变强”，容易被追问到机制细节。

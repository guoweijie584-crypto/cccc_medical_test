# docs_forHuman 使用说明

## 这个目录现在怎么读

这个目录现在不再是“所有文件堆在一层”的状态。

它分成了 3 个层次：

- `跨阶段 canonical 内容`
- `按汇报阶段进入的入口页`
- `第一次汇报直接交付物`

你可以按“我要做什么”进入，也可以按“这句话有多稳”进入。

## 一、最快的进入方式

### 1. 你想先理解整个项目

按这个顺序看：

1. `资料分类与阅读顺序.md`
2. `项目梳理与理解.md`
3. `系统设计与模块设计.md`
4. `阶段进度与里程碑.md`
5. `../docs_shared/证据边界与事实口径.md`

### 2. 你想按汇报阶段进入

按这个顺序看：

1. `汇报阶段总览.md`
2. `阶段入口/第一次汇报.md`
3. `阶段入口/第二次汇报准备.md`
4. `阶段入口/最终汇报准备.md`

### 3. 你想直接做第一次汇报

优先看：

1. `PPT生成提示词.md`
2. `超精简6分钟汇报版.md`
3. `汇报辅助/`

### 4. 你想先判断一句话能不能讲重

先看：

1. `../docs_shared/证据边界与事实口径.md`

## 二、目录内文件怎么分工

### A. 跨阶段 canonical 内容

这些文件是整个项目反复复用的主干，不属于某一轮汇报专属：

- `项目梳理与理解.md`
- `系统设计与模块设计.md`
- `阶段进度与里程碑.md`
- `资料分类与阅读顺序.md`

它们回答的是：

- 这个项目到底在做什么
- 系统怎么设计
- 现在做到哪一步
- 原始材料怎么分类

### B. 阶段入口层

这些文件只负责导读，不复制正文：

- `汇报阶段总览.md`
- `阶段入口/第一次汇报.md`
- `阶段入口/第二次汇报准备.md`
- `阶段入口/最终汇报准备.md`

当前要明确区分成熟度：

- `第一次汇报`：材料最完整，已经能直接落 PPT 或口播
- `第二次汇报准备`：现在是准备区，不是已经成型的第二版主稿
- `最终汇报准备`：现在也是准备区，不是假装已有最终稿

### C. 第一次汇报直接交付物

这些文件是当前最接近“拿来就用”的：

- `PPT生成提示词.md`
- `超精简6分钟汇报版.md`

如果还要细化口播、改原页文案、做页面微调，再去看：

- `汇报辅助/明日汇报PPT建议稿.md`
- `汇报辅助/6分钟汇报口播提纲.md`
- `汇报辅助/PPT原页修改清单.md`

### D. 共享证据边界

这份文件不是附属说明，而是整个目录的写法约束：

- `../docs_shared/证据边界与事实口径.md`

## 三、除了按用途进，还可以按证据强弱进

如果你更关心“哪个材料最稳”，现在可以这样理解：

- `主项目理解`：`项目梳理与理解.md`
- `系统设计主文档`：`系统设计与模块设计.md`
- `阶段推进主文档`：`阶段进度与里程碑.md`
- `第一次汇报直接交付`：`PPT生成提示词.md`、`超精简6分钟汇报版.md`
- `共享事实边界`：`../docs_shared/证据边界与事实口径.md`

进一步往原始依据追时，再去看：

- `3.26-SR-14.11.*`
- `archive/round1_baseline/流程架构.pptx`
- `archive/requirements_and_constraints/Multi Agent框架-0326.pdf`
- `tmp/source_repo/cccc_medical_test/`

## 四、最短建议

如果你只记 5 个入口，记这 5 个：

1. `资料分类与阅读顺序.md`
2. `汇报阶段总览.md`
3. `项目梳理与理解.md`
4. `PPT生成提示词.md`
5. `../docs_shared/证据边界与事实口径.md`

## 五、一句话收口

`docs_forHuman` 现在不是三套阶段正文，而是一套跨阶段主干 + 一层阶段入口 + 一组第一次汇报直接交付物。
